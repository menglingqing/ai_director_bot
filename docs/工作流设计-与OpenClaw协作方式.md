# 工作流设计：与 OpenClaw 的协作方式

> 所有工作都在**与 OpenClaw 机器人/助手的对话**中完成。助手负责按阶段生成提示词或直接生成图/视频；你负责在外部使用提示词，或审核助手产出的图/视频，审核通过后再进入下一步。

---

## 一、两种协作模式（由你选择）

在任意阶段，你可以选择下面两种方式之一（也可以在不同阶段用不同方式）：

| 模式 | 说明 | 你的操作 |
|------|------|----------|
| **A. 仅要提示词** | 助手只生成提示词，不直接出图/出片 | 你拿提示词到其他网站（即梦、Midjourney、Seedance 等）自己生成图或视频；需要时把结果发回给助手（如垫图、继续下一步），或口头说「我生好了，继续」 |
| **B. 助手生成 + 你审核** | 助手调用生成能力产出图或视频 | 助手给出成片/成图 → 你审核 → **通过**则助手自动进入下一步；**不通过**则你说明修改意见，助手改提示词或重生成，直到通过 |

---

## 二、审核通过后才进行下一步

- 凡是会产生**图片**或**视频**的步骤（如分镜概念图、服化道参考图、定稿分镜图、最终视频），都设一个**审核闸门**：
  - 你明确表示「通过」「可以」「进入下一步」后，助手才把该步骤视为完成，并进入下一阶段。
  - 若你说「不通过」「要改」，助手会根据你的意见修改或重生成，再次提交给你审核，直到通过。
- 仅产出**文字/提示词**的步骤（如剧本、分镜表、运镜提示词），一般不强制审核闸门，但你可以随时说「这段要改」，助手会按你的要求修改后再继续。

---

## 三、在对话中完成的事（助手侧）

助手在对话中会：

1. **按阶段推进**：根据当前阶段（剧本 → 分镜表 → 分镜概念 → 服化道 → 定稿分镜 → 运镜/卡点 → 视频）调用对应技能。
2. **产出并交付**：
   - **模式 A**：给出可直接复制使用的**提示词**（必要时注明用于哪个平台、哪个功能），方便你到其他网站生成。
   - **模式 B**：在有能力的前提下，直接产出**图/视频**，并附简短说明（如镜号、用途），供你审核。
3. **等待你的指令**：
   - 若选择模式 B，产出后明确询问：「请审核，通过请说『通过』或『进入下一步』，需要修改请说明具体意见。」
   - 收到「通过」后，再执行下一阶段；收到修改意见则在本阶段内迭代。
4. **记录上下文**：在对话中记住当前阶段、已有产物（剧本、分镜表、某镜的图等），便于你随时说「继续」「第 3 镜重做」「先出运镜提示词」等，助手都能正确衔接。

---

## 四、在对话中完成的事（你侧）

你只需要在对话里：

- **提出创意或需求**：例如「我想做一个 xxx 主题的短片」。
- **选择模式**：例如「这一阶段你只出提示词，我自己去即梦生成」或「这一阶段你直接出图，我审核通过再下一步」。
- **审核并决策**：对助手给出的图/视频回复「通过」或「不通过，希望 xxx」。
- **推进或回溯**：说「继续」「先做分镜表」「第 2 镜重做」等，助手按你的指令推进或回到某一步。

---

## 五、阶段与产出、审核点一览

| 阶段 | 主要产出 | 是否可能有图/视频 | 审核闸门（模式 B） |
|------|----------|-------------------|---------------------|
| S0 剧本 | 剧本/剧本提示词 | 否 | — |
| S1 分镜表 | 分镜表（结构化） | 否 | — |
| S2 分镜概念 | 分镜概念图提示词 或 分镜概念图 | 是 | 每镜或整批图审核通过再进 S3 |
| S3 服化道与定稿分镜 | 服化道提示词/图、定稿分镜图提示词/图 | 是 | 服化道图、定稿分镜图审核通过再进 S4 |
| S4 运镜/卡点 | 运镜提示词、卡点提示词 | 否 | — |
| S5 成片/迭代 | 视频生成提示词 或 成片视频；单镜修改提示词；画风提示词 | 是（若出片） | 成片视频审核通过即完成 |

---

## 六、与 OpenClaw 对话时的约定（给助手用的说明）

- **入口**：用户说「开始做视频」「按导演助手流程来」等，即进入本工作流；用户可在任意时刻说「只出提示词」或「你生成我来审」以选定当前阶段的模式。
- **阶段推进**：仅在用户明确说「通过」「进入下一步」或等价表达后，才将当前阶段标记为完成并进入下一阶段；若用户说「重做」「修改」，则在本阶段内迭代。
- **产物交付**：
  - 仅提示词时：输出完整、可复制的提示词，并注明建议使用的平台（如即梦、Seedance）。
  - 生成图/视频时：输出图/视频后，主动请求用户审核，并说明「通过则进入下一步，不通过请说明修改意见」。

以上约定可写入 OpenClaw 的 system prompt 或技能说明，使助手在对话中一致地按「提示词 or 生成 + 审核 → 通过再下一步」执行。
